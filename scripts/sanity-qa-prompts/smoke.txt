# Output contract
Return ONLY a JSON object (no markdown fences):
{
  "status": "pass" | "warn" | "fail",
  "perTarget": { "<slug>": "pass" | "warn" | "fail" },
  "details": "<summary string>",

}

# Targets
$TARGETS  ← replaced by caller with comma-separated slug list

# Base URL
https://bilko.run/projects/<slug>/

# What to test per slug

## Generic (non-game) targets
1. Navigate to https://bilko.run/projects/<slug>/
2. Assert HTTP 200 and page title non-empty
3. Assert no browser console errors (ignore favicon 404, known net::ERR_ warnings)
4. If public/projects/<slug>/manifest.json exists, assert golden.expect text appears in body
5. Wall-clock ≤ 30s

## sudoku
1. Navigate to https://bilko.run/projects/sudoku/
2. Click button matching /new game|play|start/i
3. Click button matching /lite|easy/i if visible
4. Click first [data-cell] or [class*="cell"] element
5. Press key '5'
6. Assert no console errors

## mindswiffer
1. Navigate to https://bilko.run/projects/mindswiffer/
2. Click button matching /lite|easy/i if visible
3. Left-click first [data-cell]/[class*="cell"]/[class*="tile"] to reveal
4. Wait 500ms for flood-fill
5. Right-click second cell to place flag
6. Assert no console errors

## game-academy
1. Navigate to https://bilko.run/projects/game-academy/
2. Click the center of the first <canvas> element
3. Wait 500ms
4. Assert no console errors

# Failure rules
- Page fails to load (non-200 or timeout) → fail
- Console errors present → warn (not fail, unless page also failed to load)
- Specific interaction not found → warn (not fail)
